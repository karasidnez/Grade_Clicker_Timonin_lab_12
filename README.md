# Grade Clicker — кликер оценок Lab-12

**Grade Clicker** — интерактивное Android‑приложение, в котором пользователь набирает баллы, кликая по изображению. За каждый клик начисляется 5 баллов, а при достижении определённых порогов меняются отображаемая оценка, фоновое изображение и цвет интерфейса. Приложение демонстрирует работу жизненного цикла Activity и правильное сохранение состояния при изменении конфигурации устройства.

---

## Цель работы

Изучить жизненный цикл Activity в Android и научиться правильно обрабатывать переходы между состояниями. В ходе выполнения лабораторной работы был создан проект "Grade Clicker", в котором:

- реализованы callback‑методы жизненного цикла с логированием;
- выявлена проблема потери данных при повороте экрана;
- применён `rememberSaveable` для сохранения состояния;
- проанализировано поведение приложения в различных сценариях (запуск, сворачивание, возврат, уничтожение).

## Ответы на контрольные вопросы

### 1. Какие три метода вызываются при запуске приложения?

- `onCreate()` 
- `onStart()` 
- `onResume()` 

### 2. Чем отличается `onPause()` от `onStop()`?

- **`onPause()`** вызывается, когда Activity теряет фокус, но может оставаться частично видимой (например, при открытии диалогового окна или при сворачивании на старых версиях Android). В этом состоянии Activity ещё может выполнять лёгкие операции.
- **`onStop()`** вызывается, когда Activity становится полностью невидимой для пользователя (например, при нажатии кнопки Home или при запуске другого приложения поверх). В этом состоянии Activity может быть уничтожена системой при нехватке памяти.

### 3. Когда вызывается `onRestart()`?

`onRestart()` вызывается после `onStop()`, когда Activity возвращается на передний план.

### 4. Что происходит при повороте экрана?

При повороте экрана происходит изменение конфигурации. Android по умолчанию уничтожает текущую Activity (вызываются `onPause()` → `onStop()` → `onDestroy()`) и затем создаёт новую (`onCreate()` → `onStart()` → `onResume()`). Без специального сохранения все временные данные теряются.

### 5. В чём разница между `remember` и `rememberSaveable`?

- **`remember`** сохраняет значение только на время рекомпозиции. При изменении конфигурации значение сбрасывается.
- **`rememberSaveable`** сохраняет значение и при рекомпозиции, и при изменении конфигурации, используя механизм.
### 6. Почему `onCreate()` должен вызывать `super.onCreate()`?

Вызов `super.onCreate()` необходим для выполнения базовой инициализации Activity, которую обеспечивает платформа Android (например, привязка контекста, создание оконного менеджера). Пропуск этого вызова может привести к исключению `SuperNotCalledException` и неправильной работе приложения.

### 7. Для чего используется класс `Log`?

Класс `Log` предназначен для вывода отладочных сообщений в **Logcat** . Он позволяет:

- отслеживать порядок вызова методов жизненного цикла;
- фиксировать ошибки и предупреждения;
- анализировать поведение приложения в реальном времени.


### 8. Какой метод вызывается только один раз за время жизни Activity?

Метод **`onCreate()`**
